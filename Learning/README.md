**学习时间**: 3 天

---

## 📚 学习路线（3 Days）

### 🟦 Day 1: RAG 核心概念（4 小时）
**目标**: 理解 RAG 的基本原理和关键参数

**学习内容**:
1. RAG 的最小结构（6 步工作流）
2. 5 个必须理解的点：
   - Chunk Size 怎么影响效果
   - Embedding 模型选择
   - Top-K 对幻觉的影响
   - RAG 能解决什么/不能解决什么
   - 什么时候不该用 RAG

**学习方式**:
- 阅读: [RAG_Day1_Core_Concepts.md](RAG_Day1_Core_Concepts.md)
- 运行: `python 25_12_22.py`（包含所有概念的可视化演示）

**验证标准**:
- [ ] 能用 5 分钟解释 RAG 给不懂的人
- [ ] 知道为什么 Chunk Size 很关键
- [ ] 能判断一个问题是否适合 RAG

**时间分配**:
- 理论学习: 2 小时
- 代码演示: 1.5 小时
- 自测题: 0.5 小时

---

### 🟦 Day 2: 业务适配（2-3 小时）
**目标**: 理解不同行业的 RAG 差异，学会诊断业务需求

**学习内容**:
1. 5 个问题诊断法（FDE 的救命技能）
   - 文档在哪里？格式？
   - 更新频率？
   - 谁用？（员工 vs 客户）
   - 错一次能不能接受？
   - 是否要可追溯？

2. 5 种公司类型的 RAG 差异
   - SaaS（权限隔离）
   - 制造业（图纸识别）
   - 法务（精确溯源）
   - 医疗（低幻觉）
   - 内部知识库（高频更新）

**学习方式**:
- 阅读: [RAG_Day2_Business_Adaptation.md](RAG_Day2_Business_Adaptation.md)
- 自测: 用 5 个问题诊断 5 个不同的被投公司

**验证标准**:
- [ ] 能快速判断一个项目的复杂度（⭐ 评分）
- [ ] 知道不同行业的 RAG 核心挑战
- [ ] 能指出"这个问题不适合 RAG"

**时间分配**:
- 理论学习: 1.5 小时
- 案例分析: 1 小时
- 自测: 0.5 小时

---

### 🟦 Day 3: 实战 Demo（4-6 小时）
**目标**: 能做一个 5 分钟的 Demo，让创始人说"这个我们能用"

**学习内容**:
1. Demo 的标准结构
   - 原始问题设定
   - 无 RAG 演示（泛答案）
   - 有 RAG 演示（准确答案）
   - 一句总结

2. 可运行的 Demo 代码
   - 文档加载
   - 分块处理
   - 检索实现
   - 结果展示

3. 常见的 5 个创始人提问

**学习方式**:
- 阅读: [RAG_Day3_Demo_Guide.md](RAG_Day3_Demo_Guide.md)
- 运行: `python RAG_Day3_Demo.py`
- 修改并尝试用自己的文档

**验证标准**:
- [ ] Demo 代码能跑通
- [ ] 能展示"无 RAG vs 有 RAG"的差异
- [ ] 能回答创始人可能问的 5 个问题

**时间分配**:
- 理论学习: 1 小时
- 代码运行和理解: 1.5 小时
- 修改和测试: 1.5 小时
- 准备演讲稿: 1 小时

## ❓ 常见问题

### Q1: 我只有 2 小时，应该学什么？
```
答: 看 RAG_Day3_Demo_Guide.md (15 分钟)
   运行 python RAG_Day3_Demo.py (15 分钟)
   理解核心代码 (30 分钟)
   
这样你能基本理解 RAG 的价值，能给非技术人士讲清楚。
```

### Q2: 代码的相似度计算为什么这么简单？
```
答: Day 1 和 Day 3 的代码是教学版，用关键词匹配而不是真实 Embedding。
   
原因:
1. 易于理解逻辑（不需要理解向量数学）
2. 快速演示（不需要调用 LLM API）
3. 完全可跑（不需要额外依赖）

实战中会用真实的 Embedding 模型。
```

### Q3: 我有真实的文档，应该怎么用？
```
答: 修改 RAG_Day3_Demo.py 中的 SampleDataGenerator 或 DocumentLoader

步骤:
1. 把你的文档放在 sample_docs/ 目录
2. 修改 load_markdown_files 的路径
3. 如果是 PDF，先用 PyPDF2 或 pdfplumber 提取文本
4. 修改分块大小，看哪个效果最好
```

### Q4: 我是非技术人士，能学这个吗？
```
答: 完全可以！

建议步骤:
1. 跳过所有代码细节
2. 读 RAG_Day1_Core_Concepts.md 的"核心概念"部分
3. 重点学 RAG_Day2_Business_Adaptation.md（这是你的工作）
4. 让技术人员跑 Demo，你关注业务诊断和客户沟通
```

### Q5: 这个学习需要什么前置知识？
```
答: 没有特殊要求，但以下知识有帮助：

必需:
- 知道什么是 AI/LLM
- 知道什么是向量（概念即可）

加分项:
- 了解 Python（理解代码）
- 了解数据库（理解向量库）
- 有 ToB 销售或实施经验（理解业务）

不需要:
- 深度学习知识
- 算法专业知识
- 数学推导
```
